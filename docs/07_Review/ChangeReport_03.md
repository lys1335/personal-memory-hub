## 详细变更说明

### 变更 1：01 记忆类型章节增加与 03 MemoryNode Level 的映射说明

**修改内容**：

- 在"4. 记忆类型体系"开头增加说明段落，指出本章是宏观分类视角，数据建模层面参见 03。
- Objective Memory → 对应 **L1: Observation**
- Knowledge Memory → 对应 **L2: Pattern** 或 **L4: State**
- Cognitive Memory → 对应 **L3: Belief**

**修改原因**：

- 01 定义的"记忆类型"与 03 的"MemoryNode Level"是同一概念的不同抽象层级，读者容易产生混淆。
- 增加映射说明后，读者可以理解三层文档之间的关联关系。

**影响范围**：

- 01 第 4 章全文。
- 不涉及任何运行时行为或数据结构变更。

**是否破坏兼容性**：❌ 否。纯文档层面的澄清。

---

### 变更 2：01 移除三类记忆的冗余 JSON 数据结构示例

**修改内容**：

- 移除了 Objective Memory、Knowledge Memory、Cognitive Memory 三个小节中的 JSON 数据结构示例。
- 替换为"数据映射"一行，指向 03 的 MemoryNode Level。

**修改原因**：

- 03 已定义 MemoryNode 层级模型，01 中的 JSON 示例与 03 的 MemoryNode Level 模型存在概念重叠。
- 避免两份文档在同一概念上给出不同视角的描述导致读者困惑。
- 具体的数据结构将在 Supabase Schema 设计阶段统一处理。

**影响范围**：

- 01 第 4.2、4.3、4.4 节。
- 如果其他文档引用了这些 JSON 示例，需要同步更新。

**是否破坏兼容性**：⚠️ 轻微。如果后续有代码或配置直接引用这些 JSON 结构，需要同步调整。但目前这些 JSON 仅为文档示例，不构成 API 契约。
